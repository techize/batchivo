## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline badge padding |
| sm | 8px | Badge gap spacing, compact row padding |
| md | 16px | Card content padding, input inner spacing |
| lg | 24px | Card header padding, section vertical rhythm |
| xl | 32px | Page header bottom margin, filter-to-list gap |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom padding (desktop) |

Exceptions:
- Touch targets for the `has_sample` icon toggle: minimum 44px hit area (accessibility). Achieved via `p-2` wrapper around a 24px icon, giving a 40px touch target minimum — use `min-w-[44px] min-h-[44px]` explicitly on the toggle button.
- Color swatch dot in row/card: 12px circle (`w-3 h-3 rounded-full`) — intentional exception below xs scale.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(40 100% 97%)` — cream `#fffaf0` | Page background (`--background`) |
| Secondary (30%) | `hsl(0 0% 100%)` — white | Cards (`--card`), Sheet background, Table rows, Filter panel |
| Accent (10%) | `hsl(32 95% 44%)` — amber `#d97706` | Focus rings (`--ring`), active filter toggle buttons, `has_sample` toggle when active |
| Destructive | `hsl(0 89% 68%)` — coral `#f56565` | Destructive actions only — none in Phase 2 |

Accent reserved for:
1. Active state of filter toggle buttons ("Needs labels", "No sample") when selected — `variant="default"` (navy fill) matching existing "Low Stock Only" button pattern
2. `has_sample` toggle button active state — amber icon / filled indicator
3. Focus ring on inputs and interactive elements

Badge color semantics for this phase:
- Spool count badge: `variant="secondary"` (warm gray)
- Labeled count badge: `variant="secondary"` when all labeled, `variant="warning"` (`amber`) when any unlabeled (e.g., "1/3 labeled")
- Sample status badge: `variant="success"` (green, using existing Badge variants) for "Sample ✓", `variant="outline"` for "Sample ✗"
- Active filter count badge on Filters button: navy pill with white numeral — use `Badge variant="default"`

Source: `frontend/src/index.css` CSS custom properties, `SpoolCard.tsx` Badge usage patterns.

Dark mode: CSS variables automatically invert via `.dark` class in `index.css`. No additional dark-mode-specific overrides needed for Phase 2 components — follow existing token usage.

## Interaction Contract

### Row Click (Opens Sheet)
- **Target**: Entire row / card, except the `has_sample` toggle area
- **Trigger**: `onClick` on row/card container; `stopPropagation()` on toggle button
- **Behavior**: Opens `FilamentTypeSpoolSheet` via shadcn `Sheet` (from right, full-height on mobile, 480px wide on desktop)
- **Loading**: Fetch `GET /api/v1/filament-types/{id}/spools` on sheet open (not prefetched). Show `Skeleton` rows while loading.

### `has_sample` Toggle (Inline on Row)
- **Component**: `Button variant="ghost" size="icon"` wrapping a Lucide `FlaskConical` icon (or `TestTube2`)
- **Active state**: amber/teal-tinted icon (`text-amber-600`) when `has_sample=true`; muted icon (`text-muted-foreground`) when false
- **Hit area**: `min-w-[44px] min-h-[44px]` to prevent accidental activation while scrolling
- **Mutation**: `PATCH /api/v1/filament-types/{id}` with `{ has_sample: !current }`. Optimistic update via TanStack Query.
- **Tooltip**: `Tooltip` component showing "Sample printed" or "Mark sample printed"

### Filter Toggle Buttons ("Needs labels" / "No sample")
- **Pattern**: Matches existing "Low Stock Only" button in `SpoolList.tsx`
- **Active**: `variant="default"` (navy fill, white text)
- **Inactive**: `variant="outline"`
- **Placement**: Row of toggle buttons below the search box, above the Filters button

### Filters Button
- **Placement**: Inline with search box in page header row
- **Active filter count**: `Badge variant="default"` overlaid on button (top-right) when any non-search filter is active
- **Sheet**: Opens `FilamentTypeFilterSheet` from right side

### Color Swatch
- **When `color_hex` present**: 12px filled circle (`w-3 h-3 rounded-full inline-block`) using `style={{ backgroundColor: "#" + normalizedHex }}`
- **When absent**: Text-only, no empty circle placeholder
- **Pattern**: Follows `SpoolCard.tsx` color_hex normalization logic (handles 6-char and 8-char hex with `0x` prefix)

### Mobile Breakpoint
- `< lg` (1024px): `FilamentTypeCard` list, no table
- `≥ lg` (1024px): Table layout (`hidden lg:block` / `lg:hidden` pattern from `SpoolList.tsx`)

### Sheet: Two Separate Instances
- `FilamentTypeSpoolSheet` — triggered by row/card click
- `FilamentTypeFilterSheet` — triggered by Filters button
- Both use `Sheet` from right. Do NOT nest sheets — use independent state variables (`spoolSheetOpen`, `filterSheetOpen`).

## Accessibility Notes

- `has_sample` toggle button must have `aria-label` set dynamically: `"Mark sample printed for {brand} {color}"` / `"Sample printed for {brand} {color} — click to unmark"`.
- Sheet must trap focus when open. shadcn Sheet handles this via Radix `Dialog` primitives.
- Color swatch dots are decorative — `aria-hidden="true"`. Color name is present as adjacent text.
- Filter toggle buttons use `aria-pressed` boolean to communicate state to screen readers.
- Table rows use `role="button"` or `tabIndex={0}` + keyboard `Enter`/`Space` handler if not using `<button>` wrapper.
- Empty and error states must be rendered in the DOM (not just visually hidden) so screen readers announce them.
