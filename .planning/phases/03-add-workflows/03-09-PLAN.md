<objective>
Close the major UAT gap where bulk and batch submit appear to succeed but spools never appear in the library. Root cause is two-part: the PostgreSQL RLS context (`SET LOCAL app.current_tenant_id`) is cleared by `db.rollback()` in the IntegrityError retry paths, causing the subsequent flush to fail the RLS INSERT check silently; and the frontend query cache for the spool sheet is not invalidated after a successful mutation.

Purpose: Spools created via bulk or batch add must appear in the filament library immediately after the dialog closes.
Output: Fixed RLS re-establishment after rollback in both endpoints; broadened cache invalidation in both mutation hooks.
</objective>
