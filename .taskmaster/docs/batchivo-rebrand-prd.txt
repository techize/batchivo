# Batchivo Rebrand PRD

## Executive Summary

Rebrand the Nozzly platform to Batchivo - a production tracking and ecommerce platform for makers. This involves a complete rebrand including code, visual identity, and deployment infrastructure.

## Background

Nozzly started as a 3D print tracking system but has evolved into a comprehensive maker business platform covering:
- Ecommerce & order management
- Production tracking (3D printing, knitting, any craft)
- Shipping & fulfillment
- Billing & invoicing
- Multi-tenant architecture
- Inventory management

The name "Nozzly" no longer reflects this broader scope. "Batchivo" (from "batch" - production batches) better represents the platform's focus on tracking production batches across all maker verticals.

## Goals

1. Complete code rebrand from "nozzly" to "batchivo"
2. Implement new visual identity (colors, typography, logo)
3. Deploy to production infrastructure
4. Maintain zero downtime for existing nozzly.app users during transition

## Non-Goals

- Feature changes or new functionality (separate PRD)
- Database migration (same schema, different branding)
- User migration (handled separately)

## Phase 4: Deployment

### 4.1 Kubernetes (k3s) Deployment

**Namespace:** batchivo (new namespace)

**Services to deploy:**
- batchivo-backend (FastAPI)
- batchivo-frontend (Nginx + React build)
- batchivo-worker (Celery)

**Shared services (existing):**
- PostgreSQL (new database: batchivo)
- Redis (existing cluster)
- MinIO (new bucket: batchivo)

### 4.2 Cloudflare Tunnel

**Existing tunnel:** (same as nozzly/mystmereforge)

**New public hostnames:**
- batchivo.com -> batchivo-frontend service
- api.batchivo.com -> batchivo-backend service
- dev.batchivo.com -> dev environment

### 4.3 Environment Configuration

**Required secrets:**
- DATABASE_URL
- REDIS_URL
- JWT_SECRET
- SQUARE_API_KEY (if using Square payments)
- S3/MinIO credentials
- SMTP credentials
